## 使用Helm部署dashboard

- 参考地址

  ###### [Kubernetes系列之Helm介绍篇](https://www.cnblogs.com/guigujun/p/10550256.html)

  

- 查看helm仓库

  ```shell
  helm repo list
  ```

- 添加国内源

  ```shell
  helm repo add stable https://kubernetes.oss-cn-hangzhou.aliyuncs.com/charts
  ```

- 更新仓库

  ```shell
  helm repo update
  ```

- 拉取kubernetes-dashboard

  ```shell
  helm fetch stable/kubernetes-dashboard
  ```

  ```shell
  [root@k8s-master01 helm-dashboard]# ls
  kubernetes-dashboard-0.6.0.tgz
  ```

  ```
  tar -zxvf kubernetes-dashboard-0.6.0.tgz
  ```

## [kubernetes搭建dashboard报错](https://www.cnblogs.com/xulingjie/p/10101321.html)

- 1.添加serviceaccount账户，设置并使其可登陆

  ```
  apiVersion: v1
  kind: ServiceAccount
  metadata:
    name: aks-dashboard-admin
    namespace: kube-system
  ---
  apiVersion: rbac.authorization.k8s.io/v1
  kind: ClusterRoleBinding
  metadata:
    name: aks-dashboard-admin
  roleRef:
    apiGroup: rbac.authorization.k8s.io
    kind: ClusterRole
    name: cluster-admin
  subjects:
  - kind: ServiceAccount
    name: aks-dashboard-admin
    namespace: kube-system
  ```

- 2.创建完全管理权限

  ```
  apiVersion: rbac.authorization.k8s.io/v1beta1
  kind: ClusterRoleBinding
  metadata:
    name: kubernetes-dashboard
    labels:
      k8s-app: kubernetes-dashboard
  roleRef:
    apiGroup: rbac.authorization.k8s.io
    kind: ClusterRole
    name: cluster-admin
  subjects:
  - kind: ServiceAccount
    name: kubernetes-dashboard
    namespace: kube-system
  ```

  ```
  apiVersion: rbac.authorization.k8s.io/v1beta1
  kind: ClusterRoleBinding
  metadata:
    name: kubernetes-dashboard-head
    labels:
      k8s-app: kubernetes-dashboard-head
  roleRef:
    apiGroup: rbac.authorization.k8s.io
    kind: ClusterRole
    name: cluster-admin
  subjects:
  - kind: ServiceAccount
    name: kubernetes-dashboard-head
    namespace: kube-system
  ```
